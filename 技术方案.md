## 一、方案概述

### 1.1 目标

构建面向投研投顾直播间场景的**多聊天室群聊系统**：

- 多聊天室并行管理（CRUD、状态流转、成员管理）
- 管理员广播；普通用户发言**经审核通过后**才展示
- 仅群组交流，无私聊
- WebSocket 实时推送；审核异步 + 超时自动拒绝
- RBAC 权限、全局配置、性能监控
- 目标：**1000 并发连接**、**峰值 100 条/秒**

### 1.2 技术选型

| 层次 | 技术 | 说明 |
|------|------|------|
| 后端 | Spring Boot 3.2 · JDK 21 | REST + WebSocket |
| 实时通信 | STOMP over WebSocket | 单连接多路复用 |
| ORM | MyBatis-Plus 3.5 | 逻辑删除、乐观锁、分页 |
| 数据库 | MySQL 8 / PostgreSQL 14+ | Flyway `{vendor}` 双库支持 |
| 缓存 | Redis 6+ · Guava | 队列、登录态、二级缓存 |
| 认证 | JWT (jjwt) | 无状态 + minIat 强制失效 |
| 前端 | Vue 3 + Vite + Pinia + Element Plus | 动态菜单 + 路由守卫 |
| 监控 | Micrometer + Prometheus | 仪表盘 + Actuator |
| 测试 | JUnit 5 + Mockito + JaCoCo | 行覆盖率 ≥ 80% |

### 1.3 关键设计决策

1. **STOMP 单连接多路复用**，订阅 `/topic/room.{roomId}`，降低连接数。
2. **审核队列 Redis（List + ZSet）**，消息体存 DB，Redis 只存 ID。
3. **审核无状态可扩展**；多实例推送可接 Redis Pub/Sub 或 STOMP Relay。
4. **乐观锁 + CAS** 保证审核/超时并发安全；**事务提交后推送**。
5. **at-least-once 推送** + 客户端 `messageId` 去重。
6. **RBAC 菜单路径鉴权**，支持自定义角色灵活授权。
7. **在线人数**统计登录态（Redis），非 WebSocket 连接数。

## 二、系统架构

### 2.1 分层架构

```
前端 Vue 3 SPA
    │ REST (JWT)          │ WebSocket/STOMP (JWT)
    ▼                     ▼
Controller / WS 端点 + 拦截器鉴权
    ▼
Service（Auth / Room / Member / Message / Audit / Push / RBAC / Config / Monitor）
    ▼
Mapper（MyBatis-Plus）
    ├─ Redis：审核队列、登录态、WS 会话、权限/配置缓存
    └─ MySQL / PostgreSQL：业务持久化
```

### 2.2 核心模块

| 模块 | 职责 |
|------|------|
| AuthService | 注册、登录、退出；登录态写入 Redis |
| ChatRoomService | 聊天室 CRUD、状态、列表 |
| MemberService | 加入/退出/审批/拉人/踢人/候选人搜索 |
| MessageService | 提交、历史、广播、VO 转换 |
| AuditService | 审核、批量、超时、审计日志 |
| PushService | STOMP 推送（异步、重试） |
| MenuPermissionService | RBAC 菜单路径校验 |
| SysConfigService | 全局配置 + 二级缓存 |
| MonitorService | 监控指标聚合 |
| PermissionCache / TwoLevelCache | 高并发读优化 |

### 2.3 部署拓扑

```
Nginx / LB
    ├── App 实例 1..N（无状态 Spring Boot）
    ├── MySQL / PostgreSQL
    └── Redis（队列 + 缓存 + Pub/Sub）
```

## 四、WebSocket 实现

完整协议见 **[docs/接口与WebSocket协议说明.md](./docs/接口与WebSocket协议说明.md)** 第三节。

| 目的地 | 用途 |
|--------|------|
| `/ws?token=` | SockJS + STOMP 端点 |
| `/topic/room.{roomId}` | 已通过聊天消息 |
| `/user/queue/notifications` | 个人通知 |
| `/topic/audit` | 待审核提醒 |

**优化要点**（详见架构文档）：

- 前端全局 WS 单例，避免页面切换断连
- CONNECT 帧写入会话；SUBSCRIBE 仅对敏感 Topic 鉴权
- 重连自动恢复订阅 + 历史 REST 补齐

## 五、审核流程

```
提交 → messages(PENDING) → Redis 队列
    → 管理员审核（锁 + CAS）
    → 事务提交 → 异步 pushToRoom / notifyUser
    → 超时任务扫描 ZSet → TIMEOUT + 通知
```

## 六、REST API

共 **44** 个接口，分认证、聊天室、成员、消息、审核、配置、监控、系统管理（用户/角色/菜单）等模块。

完整路径、请求体、权限见 **[docs/接口与WebSocket协议说明.md](./docs/接口与WebSocket协议说明.md)**。

## 七、前端设计

| 页面 | 路由 | 说明 |
|------|------|------|
| 登录/注册 | `/login` | JWT 登录 |
| 工作台 | `/app/dashboard` | 默认落地页 |
| 聊天室列表 | `/app/rooms` | 浏览、管理 |
| 聊天室 | `/app/rooms/:id` | 实时消息 + 提交 |
| 我的消息 | `/app/my-messages` | 审核状态 |
| 消息广播 | `/app/broadcast` | 多房间广播 |
| 消息审核 | `/app/audit` | 审核面板 |
| 性能监控 | `/app/metrics` | 在线人数、延迟等 |
| 全局配置 | `/app/configs` | 运行参数 |

- 路由守卫按菜单路径鉴权
- `stores/ws.js` 全局 WebSocket；`stores/auth.js` 登录态

## 八、性能、监控与运维

| 需求 | 方案 |
|------|------|
| 1000 连接 | STOMP 多路复用 + 横向扩展 |
| 100 msg/s | 审核/推送异步 + Redis 削峰 |
| 在线人数 | Redis 登录态计数 |
| 监控指标 | 在线人数、队列长度、P95/P99、错误率、连接池 |
| 暴露 | `/api/metrics/dashboard`、`/actuator/prometheus` |

线上问题处理：结构化日志 + TraceId → Prometheus 告警 → Grafana → 版本回滚。
